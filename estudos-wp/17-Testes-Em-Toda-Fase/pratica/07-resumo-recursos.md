# Resumo e recursos – Testes em toda fase

Fonte: **017-WordPress-Fase-17-Testes-Em-Toda-Fase.md** (Resumo).

---

## O que você aprendeu

- **Testing throughout** – Integrar testes a cada tópico (hooks, REST, CPT, arquitetura, async), não deixar testes para o fim.
- **Setup** – PHPUnit, estrutura de pastas (Unit/Integration), bootstrap WordPress para WP_UnitTestCase.
- **Testes por fase** – Hooks (actions/filters), REST (status, auth, validação, sanitização, capabilities), CPT/Settings/shortcodes, arquitetura (mocks, Repository, Service, DI), Action Scheduler.
- **Boas práticas** – Nomenclatura, AAA, testes independentes, mocking adequado, cobertura e regressão.

---

## Próximos passos

1. Implementar testes enquanto estuda cada fase (não adiar).
2. Aumentar cobertura aos poucos (começar por código crítico).
3. Refatorar código para ser testável (injeção de dependências, funções puras onde possível).
4. Aprofundar na Fase 10 (Testes, Debug e Implantação) para testes avançados (mocking, data providers, integração).
