# Comunidade (Slack, Make teams)

**Slack Make WordPress**; times (Core, Plugins, Themes, etc.); como pedir ajuda e onde reportar bugs.

---

## Slack Make WordPress

- Acessar [make.wordpress.org/chat](https://make.wordpress.org/chat/).
- Solicitar convite para o **Slack** (WordPress.org); canais por equipe e idioma (ex.: #core, #plugins, #themes, #meta, #polyglots, #br-portugues).

---

## Times (Make teams)

| Time | Foco |
|------|------|
| **Core** | Desenvolvimento do núcleo do WordPress; Trac, patches, releases. |
| **Plugins** | Diretrizes e revisão de plugins no .org; Plugin Handbook. |
| **Themes** | Temas e revisão no .org; Theme Handbook. |
| **Meta** | WordPress.org, ferramentas (SVN, APIs). |
| **Documentation** | Documentação para desenvolvedores (Developer Handbook). |
| **Accessibility** | Acessibilidade no WordPress. |
| **Polyglots** | Traduções. |

Cada time tem reuniões, blog no Make e canais no Slack.

---

## Pedir ajuda

- **Slack:** canal adequado (#plugins para dúvidas de plugin, #core para core, etc.); ser objetivo e informar versão do WP/PHP.
- **Fóruns:** [wordpress.org/support](https://wordpress.org/support/) – suporte por plugin/tema.
- **Documentação:** [developer.wordpress.org](https://developer.wordpress.org/) – Handbooks (Plugin, Theme, REST API, Block Editor).

---

## Reportar bugs

- **Core:** [core.trac.wordpress.org](https://core.trac.wordpress.org/) – criar ticket com passos para reproduzir, versão, ambiente.
- **Plugins/Themes do .org:** na página do plugin/tema, aba “Support” ou “Report”; ou no Slack #plugins/#themes.
- **Segurança:** [wordpress.org/security](https://wordpress.org/security/) – reportar vulnerabilidades de forma responsável.
